🛡️ Web Security: 25 Question Master Key

Exam Questions & Definitions

  1. Define Web Application Security? The process of protecting websites and online services against different security threats that exploit vulnerabilities in an application's code.
  2. Explain SOP (Same-Origin Policy)? A critical web security concept that restricts how a document or script loaded from one origin can interact with a resource from another origin, preventing malicious scripts on one page from obtaining access to sensitive data on another.
  3. What is The Defense-in-Depth Approach? An information security approach where multiple layers of security controls are placed throughout an IT system. If one layer fails, another layer catches the threat.
  4. Write down the difference between HTTP and HTTPS? HTTP transmits data in plain, readable text. HTTPS uses SSL/TLS encryption to secure the communication, preventing attackers from reading intercepted data.
  5. Explain NEVER TRUST THE USER? A fundamental security principle stating that all input received from a user (or outside the system's control) must be considered dangerous and strictly validated or sanitized before processing.
  6. What is Security Misconfiguration? Occurs when security settings are poorly configured, left at default, or incomplete, allowing attackers unauthorized access to system data or functionality.
  7. Explain what is Access Control? The security mechanism that dictates who (or what) can view or use resources in a computing environment, ensuring only authorized users access specific data.
  8. Explain Broken Authentication and Session Management? A vulnerability where an application improperly implements authentication and session management, allowing attackers to compromise passwords, keys, or hijack session tokens.
  9. Explain Insecure Cryptographic Storage? A flaw where sensitive data (like passwords or credit card numbers) is not properly encrypted at rest, making it easily readable to attackers who breach the database.
  10. Explain Whitelist Validation? A strict security practice where only data that matches a known-good, predefined set of allowed characters or values is accepted, immediately rejecting everything else.
  11. Explain Built-In HTTP Authentication? Mechanisms provided by the HTTP protocol itself (like Basic or Digest authentication) to challenge and verify user credentials directly via the browser and server response headers.
  12. Explain two Common Types of Brute-Force Attacks? 1) Dictionary Attack: Using a pre-compiled list of common words. 2) Credential Stuffing: Automating the injection of breached username/password pairs.
  13. Explain Application Security? The broad practice of adding features or functionality to software to prevent cyber threats, encompassing the entire software development lifecycle.
  14. Explain Rubber-Hose Attack? A cryptographic attack where the attacker extracts secrets (like cryptographic keys or passwords) from a person through physical coercion, torture, or extortion rather than technical hacking.
  15. Explain Require Minimum Password Length? A policy ensuring passwords contain a minimum number of characters to mathematically increase the time and computing power needed for an attacker to crack them.
  16. Explain Brute Force Attack? A trial-and-error method used by attackers to guess credentials or encryption keys by systematically submitting thousands of passwords until the correct one is found.
  17. Write any OWASP Top Three? Historically: 1. Injection (e.g., SQLi), 2. Broken Authentication, 3. Sensitive Data Exposure.
  18. Explain Authentication Fundamentals? The core principles of verifying a user's identity, typically relying on something you know (password), something you have (token), or something you are (biometrics).
  19. Explain Network Security? The practice of preventing and protecting against unauthorized intrusion into corporate networks, protecting the usability and integrity of network infrastructure.
  20. Explain CSS? While typically standing for Cascading Style Sheets (design), in older security contexts, it sometimes referred to Cross-Site Scripting (now universally known as XSS), where attackers inject malicious scripts into trusted websites.
  21. What are Cookies? Small pieces of data stored on the user's computer by the web browser, critical for tracking active sessions and maintaining user state (login status).
  22. Why the Strategy is Evolving? Security strategies must continuously adapt because threat actors constantly develop new attack vectors, technologies change rapidly, and network systems become increasingly complex.
  23. Explain Rotate Passwords? A security policy requiring users to change their passwords periodically to limit the lifespan and usefulness of potentially compromised credentials.
  24. Explain Password Cannot Equal Username? A basic input validation rule to prevent trivial brute-force attacks where an attacker assumes the user was lazy and used their login name as their password.
  25. Explain Common Reasons for a Disabled Account? Accounts are typically disabled automatically due to multiple failed login attempts (lockout mechanism), prolonged inactivity, or manual administrative suspension for policy violations.
