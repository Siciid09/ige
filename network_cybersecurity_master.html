Part 1: Network Security Strategies

1.1 The Two Sides of Cybersecurity

Cybersecurity is divided into two main factions: Offensive and Defensive security.

  • Offensive Security (Red Team): The Invading Army. Goal: Find a weak spot in the wall, sneak through the sewer grate, or bribe a guard before the King’s feast is stolen.
  • Defensive Security (Blue Team): The Castle Guards. Goal: Build higher walls, dig moats, train sentries to spot spies, and have a plan to lock down the throne room if an intruder is detected.

1.2 Offensive Security (OffSec)

Definition: Proactive security measures designed to identify vulnerabilities by simulating real-world attacks before malicious hackers do.

Key Objectives:

  • Find weaknesses (technical, human, physical).
  • Measure risk (how deep can an attacker go?).
  • Test incident response capabilities.
  • Avoid compliance fines and data breaches.

Offensive Security Disciplines:

  • Penetration Testing (Pentesting): Authorized simulated attack on systems (Scope defined, time-boxed).
  • Red Teaming: Goal-based, no-holds-barred simulation. Tests people, processes, and physical security (e.g., “Steal the R&D data”). Often uses stealth.
  • Vulnerability Assessment: Automated scanning to identify known CVEs (Common Vulnerabilities and Exposures).
  • Social Engineering: Exploiting human psychology (Phishing emails, vishing calls, tailgating into buildings).
  • Bug Bounty: Crowdsourced hacking; ethical hackers are paid per vulnerability found.

1.3 Defensive Security

Defensive Security Disciplines:

  • Security Operations Center (SOC): The “nerve center.” Analysts monitor logs and alerts 24/7.
  • Incident Response (IR): The “fire department.” Steps taken to contain, eradicate, and recover from a breach.
  • Threat Hunting: Proactively searching networks for threats that evaded automated detection.
  • Digital Forensics: Investigating the “crime scene” to find out how the breach happened and who did it.
  • Compliance & Governance: Ensuring the organization meets standards (ISO 27001, GDPR, HIPAA) to avoid legal liability.
Why Organizations Need Both:
Only Offensive (Red): You know where you are weak, but you have no way to stop the attack or monitor if someone is inside.
Only Defensive (Blue): You have alarms and barriers, but you don’t know if the barriers are actually strong enough to stop a determined attacker.
Purple Teaming: Bridges the gap, turning findings into actionable defense improvements. You must be able to both fight and fortify!

Part 2: Cryptography Basics

Definition: Cryptography protects data by converting it into secure formats. It is the practice and study of techniques for secure communication in the presence of adversaries (eavesdroppers). It constructs protocols that prevent public reading of private messages.
Etymology: Greek Krypto's (hidden, secret) + graphene (to write) = "hidden writing."

2.1 The Four Goals of Cryptography

  • Confidentiality (Privacy): Keeping information secret from unauthorized parties (Encryption).
  • Integrity: Ensuring that information has not been tampered with or altered during transmission or storage.
  • Authentication: Verifying the identity of the sender/receiver (who created/accessed the data).
  • Non-Repudiation: Preventing a party from denying a previous action or communication (e.g., "I didn't send that email").

2.2 Types of Cryptography

  • 1. Symmetric-Key Cryptography (Secret-Key): One shared secret key is used for BOTH encryption and decryption. Like a locked mailbox where both sender and receiver have the same key.
    Pros: Very fast, efficient for bulk data.
    Note: Modes of Operation (for block ciphers like AES - 128 bits) define how to handle longer data.
  • 2. Asymmetric-Key Cryptography (Public-Key): Uses a key pair: a public key (shared with everyone) and a private key (kept secret). Data encrypted with the public key can ONLY be decrypted with the private key.
    Pros: Solves key distribution problem. Enables digital signatures.
    Cons: Very slow (100-1000x slower than symmetric).
  • 3. Cryptographic Hash Functions: Takes any input and produces a fixed-size string of bytes (the hash/digest). It is strictly one-way (cannot reverse).
    Properties: Deterministic (same input = same hash), Fast to compute, Pre-image resistant (impossible to find input given the hash).

Applications: Banking, Communication, Cybersecurity.
Final Golden Rule (Schneier's Law): "Anyone can invent a security system that he himself cannot break."

Part 3: Key Distribution & Authentication

3.1 Key Distribution

The process of securely sharing cryptographic keys. Essential for encrypted communication.

  • Symmetric Challenge: The secure delivery of the shared key is the main vulnerability.
  • Asymmetric Solution: Supports secure key exchange since public keys can be shared openly.
  • Key Distribution Center (KDC): Trusted third party for distributing keys. Used in Kerberos. Reduces need for multiple shared keys.
  • Diffie-Hellman Key Exchange: Allows two parties to establish a shared secret over insecure channels. Foundation for many secure protocols.

3.2 User Authentication

Verifies the identity of users or systems and protects resources from unauthorized access.

Authentication Factors:

  • Something you know (password)
  • Something you have (smart card/token)
  • Something you are (biometrics)

Authentication Protocols: Kerberos, RADIUS, TACACS+, Multi-factor Authentication (MFA).

Common Security Threats:
Man-in-the-middle attacks, Replay attacks, Phishing and credential theft, Weak password vulnerabilities.

Best Practices:
Use strong encryption algorithms, Implement MFA, Regularly rotate cryptographic keys, Monitor and audit authentication logs.

Part 4: Wireless Network Security

4.1 IEEE 802.11 WLAN Overview

WLAN standard family mapping MAC and PHY layers. Channel access uses CSMA/CA. Common standards: 802.11a/b/g/n/ac/ax.

Architecture Components:

  • STA: Station (The client device)
  • AP: Access Point
  • BSS: Basic Service Set
  • ESS: Extended Service Set
  • DS: Distribution System

4.2 IEEE 802.11i Wireless LAN Security

Enhances WLAN security and introduced the Robust Security Network (RSN). This forms the basis for WPA2, providing major authentication and encryption improvements.

802.11i Security Features:

  • 802.1X authentication
  • EAP framework
  • AES-CCMP encryption
  • Key management
  • Protection against WEP weaknesses

Protocol Evolution:

  • WEP: Weak security.
  • WPA: Transitional improvement.
  • WPA2 (802.11i): Strong security.
  • WPA3: Latest enhancements.

4.3 WAP (Wireless Application Protocol) Overview

Standard for mobile web access. Optimized for wireless devices to support communication between mobile devices and web services.

  • Architecture: Mobile Device -> WAP Gateway/Proxy -> Web Server -> Wireless communication network.
  • WAP Protocol Stack:
  • WAE: Application Environment
  • WSP: Session Protocol
  • WTP: Transaction Protocol
  • WTLS: Security
  • WDP: Datagram Protocol
