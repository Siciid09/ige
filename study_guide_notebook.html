📱 Mobile App II: Final Master Notes

📌 1. Flutter & Dart Basics

  • Flutter: Google's UI toolkit for natively compiled cross-platform apps.
  • Dart: The object-oriented language behind Flutter. Compiles to native code & JS.
  • Hot Reload: Updates UI fast, keeps app state.
  • Hot Restart: Slower, completely clears the app state.

🏗️ 2. Architecture & Ecosystem

  • Framework: The Dart layer (Widgets, animation).
  • Engine: The C++ core using the Skia renderer to draw pixels.
  • Embedder: Platform integration (Android, iOS).
  • Competitors: React Native (JS), Swift (Apple), Kotlin (Android).

🧩 3. The Ultimate Widget List

  • Stateless Widgets: Never change after building (e.g., Text, Icon).
  • Stateful Widgets: Change during runtime (e.g., forms, buttons).
  • MaterialApp: Root widget. Use debugShowCheckedModeBanner: false to hide the banner.
  • Scaffold: Provides appBar and body.
  • Layouts (Single Child): Center, Container, SizedBox, Padding.
  • Layouts (Multi Child): Column, Row, Stack, ListView.
  • SingleChildScrollView: Makes screen scrollable (crucial for login pages).
  • TextField: Uses InputDecoration for styling and OutlineInputBorder.
  • Navigation: Navigator.push() goes to new screen. Navigator.pop(context) goes back.

📂 4. Project Structure & Setup

  • lib/main.dart: Main code file. Entry point is void main().
  • pubspec.yaml: File for packages (supabase_flutter), fonts, and images.
  • AndroidManifest.xml: MUST declare android.permission.INTERNET here!
  • CLI Commands: flutter doctor (checks setup), flutter pub get (downloads packages).

🗄️ 5. Supabase (The Backend)

  • Supabase: Open-source Firebase alternative using PostgreSQL.
  • Anon Key: Public key safe for the frontend app.
  • Secret Key: MUST ONLY be used on the backend!
  • RLS (Row Level Security): Policies that control who can read/write data.
  • Auth Logic: Use .insert() to register users. Use .select().eq().maybeSingle() to login.
