Chapter 1 — Introduction to Digital Signal Processing

In this modern world we are surrounded by all kinds of signals in various forms. Some are natural, but most are manmade. Some signals are necessary (like speech), while many are unwanted or unnecessary in a given situation. In an engineering context, signals are carriers of information, both useful and unwanted. Therefore extracting or enhancing the useful information from a mix of conflicting information is the simplest form of signal processing.

Signal Processing (General Definition) Signal processing is an operation designed for extracting, enhancing, storing, and transmitting useful information.

1.1 How Are Signals Processed?

The signals we encounter in practice are mostly analog signals — they vary continuously in time and amplitude. There are two approaches to processing them:

Analog Signal Processing (ASP)
  • Uses electrical networks containing active and passive circuit elements (resistors, capacitors, op-amps)
  • Processes signals in continuous time
  • Example: Radio and TV receivers
  • Prone to temperature, aging, and noise effects
  • Reconfiguration requires rewiring hardware
Digital Signal Processing (DSP)
  • Uses digital hardware: adders, multipliers, logic elements, or special-purpose microprocessors
  • Analog signals must first be converted to digital (binary numbers / bits)
  • Processing is done on discrete sequences
  • Stable, repeatable, temperature-independent
  • Easily modified via software

What each block does:
PrF (Pre-Filter / Anti-Aliasing Filter): Conditions the analog signal before sampling to prevent aliasing — i.e., it removes frequency components above half the sampling rate (Nyquist frequency) so the digital representation is accurate.

ADC (Analog-to-Digital Converter): Converts the continuous analog waveform into a stream of binary numbers (samples) at regular intervals. This is where the analog signal becomes digital.

DSP Processor: The heart of the system. Can be a general-purpose computer, a special-purpose processor, or dedicated digital hardware. Performs computations (additions, multiplications) on the digital samples.

DAC (Digital-to-Analog Converter): Inverse of the ADC. Converts the sequence of binary numbers back into a continuous (staircase) waveform — the first step toward reproducing an analog signal.

PoF (Post-Filter): Smooths out the staircase waveform produced by the DAC into the desired smooth analog output signal.

The DSP system performs these four key operations:

  1. Accepts an analog signal as input
  2. Converts this analog signal to numbers (via ADC)
  3. Performs mathematical computations on the numbers
  4. Converts the results back into an analog signal (via DAC)

1.2 Advantages of DSP over ASP

A major drawback of ASP is lack of flexibility in processing and complexity in system designs, generally leading to expensive products. Using a DSP approach, it is possible to convert an inexpensive personal computer into a powerful signal processor.

  • Flexibility / Reconfigurability: DSP systems can be developed using software on general-purpose computers. Easy to modify in real-time — often just by changing program coefficients or reloading registers — rather than rewiring hardware. Analog filter reconfiguration is very complex; digital filters just need a program update.
  • Accuracy & Stability: DSP operations are based solely on additions and multiplications, leading to extremely stable processing. Performance is independent of temperature, component ageing, and power variations. Analog circuits are prone to these external effects.
  • Cost: Lower cost due to VLSI technology, which reduces costs of memories, gates, and microprocessors.
  • Storage: Digital signals can be easily stored on magnetic media (hard drives), optical media (CDs, DVDs), or semiconductor chips, without degradation.
  • Multiplexing: Provides the foundation for ISDN (Integrated Services Digital Network) — digitized signals can be multiplexed with other digital data and transmitted through the same channel.
  • Immunity to Noise: Digital filters are highly immune to noise and relatively stable; analog circuits pick up external interference.
  • Easy Operation: Even complex mathematical operations can be performed easily using computers; not possible with analog processing.
  • Multiple Filtering: Multiple filtering is possible only with digital filters (processing one signal through several filters simultaneously).
  • Wide Variety of Response Shapes: Digital filters can achieve amplitude and phase responses that are extremely difficult or impossible to realize with analog components.
  • Impedance Matching: Impedance matching problems are minimum in digital systems.
  • Portability: The software implementing a DSP system is portable between different hardware platforms.

1.3 Filters

Looking at the DSP block diagram, the filter is the main component of any DSP system. Filters have two primary uses:

1. Signal Separation Needed when a signal has been contaminated with interference, noise, or other signals. The filter separates the desired signal from the unwanted content.
Example: A radio receiver selecting one station while rejecting all others.
2. Signal Restoration Used when a signal has been distorted in some way. The filter attempts to restore the signal to its original form.
Example: An audio recording made with poor equipment filtered to recover original sound quality; deblurring of an image from an improperly focused lens.

1.3.1 Analog Filters

  • Take analog signal as input and produce analog output
  • Constructed using resistors, capacitors, inductors, and active components (op-amps)
  • Described mathematically by differential equations
  • Analyzed using Laplace Transform
  • Frequency response modified by changing component values
  • Advantages: Cheap, large dynamic range in both amplitude and frequency
  • Disadvantages: Performance inferior to digital filters; affected by temperature, ageing; hard to reconfigure

1.3.2 Digital Filters

  • Process and generate digital data
  • Constitute elements: adder, multiplier, and delay units
  • Described mathematically by difference equations
  • Analyzed using Z-Transform
  • Frequency response changed by changing filter coefficients
  • Performance vastly superior to analog filters
Advantages of Digital Filters:
  • Performance NOT influenced by component ageing, temperature, or power variations
  • Highly immune to noise and relatively stable
  • Wide variety of shapes for amplitude and phase responses
  • Impedance matching problems are minimum
  • Transportation and reconfiguration is very easy
  • Multiple filtering is possible
  • Computational problems are minimum

1.4.1 Telecommunications — Detailed Study

Telecommunications is about transferring information from one location to another — telephone conversations, television signals, computer files, and other data. The financial principle is simple: the more information that can pass through a single channel, the more money a company makes. DSP has revolutionized telecommunications in several key areas:

A. Multiplexing
The Problem: There are approximately one billion telephones in the world. Until the 1960s, a connection between two telephones required passing analog voice signals through mechanical switches. One connection required one pair of wires — enormously expensive at scale.

The DSP Solution: DSP converts audio signals into a stream of serial digital data. Since bits can be easily interleaved and later separated, many telephone conversations can be transmitted on a single channel.

The T-Carrier System (Key Example):
• Simultaneously transmits 24 voice signals on a single channel
• Each voice signal sampled at 8,000 samples/second
• Each sample encoded with 8 bits (companded / logarithmically compressed)
• Each channel data rate: 8,000 × 8 = 64,000 bits/second = 64 kbps
• Total for all 24 channels: 24 × 64,000 = 1.544 Megabits/second (Mbps)
• Can be transmitted ~6,000 feet using ordinary 22-gauge copper telephone wire
• Financial advantage: Wire and analog switches are expensive; digital logic gates are cheap.
B. Compression
The Problem: When a voice signal is digitized at 8,000 samples/sec, most of the digital information is redundant — the information in any one sample is largely duplicated by neighboring samples.

DSP Compression Algorithms convert digitized voice into data streams requiring fewer bits/sec:
• 64 kbps → 32 kbps: No loss of sound quality (2:1 compression)
• 64 kbps → 8 kbps: Noticeably affected but still usable for long distance (8:1 compression)
• 64 kbps → 2 kbps: Highest achievable compression; highly distorted but usable for military and undersea communications (32:1 compression)

Matching decompression algorithms are used to restore the signal to its original form.
C. Echo Control
The Problem: In long-distance telephone connections, when you speak into a telephone, a portion of your signal returns as an echo. For short distances, the delay is only a few milliseconds — the ear is accustomed to this. As distance increases, the delay can reach several hundred milliseconds for intercontinental communications — extremely objectionable.

The DSP Solution: DSP attacks this by measuring the returned signal and generating an appropriate antisignal to cancel the offending echo.

The same technique:
  • Allows speakerphone users to hear and speak simultaneously without audio feedback (squealing)
  • Can reduce environmental noise by canceling it with digitally generated antinoise

1.4.2 Audio and Music Processing

In all musical recordings, sounds from instruments are recorded in a studio. Special audio effects are then added by manipulating the recorded sounds using various DSP techniques. The sound reaching listeners in a concert hall consists of direct sound, early reflections, and reverberations (echoes) — but studio recordings lack this natural quality. DSP recreates these effects.

Echo Generation (FIR Filter):
Simple single echo — direct sound + one echo K samples later:
H(z) = 1 + αz⁻ᴷ (FIR filter with delay)

Multiple echoes separated K sampling periods: use FIR filter with multiple delay taps

Infinite echoes, exponentially decaying, spaced K periods apart:
H(z) = 1 / (1 - αz⁻ᴷ) (IIR filter — recursive, infinite echo generator)
Flanging Effect:
Created by feeding the same musical note to two tape recorders and combining their delayed outputs.
Simulated using an FIR filter by periodically varying the delay K(n) between 0 and K.

Chorus Effect:
Achieved when several musicians play the same note simultaneously with small changes in amplitudes and small timing differences.
Simulated by generating multiple slightly-delayed copies of the signal with slightly different amplitudes.

1.5 Simple Architecture of a DSP Processor

A typical DSP processor has a highly optimized architecture for multiply-accumulate (MAC) operations:

  • MAC Unit: Performs multiplication and accumulation in a single clock cycle — critical for convolution and filtering
  • Program Memory: Stores instructions (coefficients)
  • Data Memory: Stores signal samples
  • Address Generation Unit: Generates memory addresses using modulo arithmetic (circular buffers)
  • I/O Ports: Interface with ADC and DAC

Signal (Definition) A measurable physical quantity which is a function of one or more independent variables.

2.1 Discrete-Time Signals

A discrete signal is denoted by x(n), where the variable n is integer-valued and represents discrete instances in time. It is also called a discrete-time signal, which is essentially a number sequence.

Note: Digital processors only take discrete-time signals. Therefore, continuous-time signals must first be converted to discrete-time using an ADC circuit.

We use several elementary sequences in DSP for two major purposes:

  1. As test signals to test the characteristics of a system
  2. Complex signals can be expressed in terms of these basic signals for analysis

2.2.1 Unit Sample (Impulse) Sequence — δ(n)

δ(n) = { 1 ,  if n = 0
         { 0 ,  if n ≠ 0
Key Properties of δ(n):
  • Sifting Property: x(n)·δ(n − k) = x(k)·δ(n − k) — "sifts out" the value x(k)
  • Unit Sample Synthesis: Any signal x(n) can be written as a weighted sum of shifted impulses:
    x(n) = Σ x(k)·δ(n − k) for k = −∞ to +∞
  • Has unit energy: E = 1
  • ROC of Z{δ(n)} = 1 is the entire Z-plane

Shifted impulse: δ(n − k) = 1 at n = k, 0 otherwise. The delay element z⁻¹ in system diagrams represents a one-sample delay, equivalent to shifting by δ(n − 1).

2.2.4 Sinusoidal Sequence — x(n) = A·cos(ω₀n + θ)

x(n) = A · cos(ω₀n + θ) where A = amplitude, ω₀ = digital frequency (rad/sample), θ = phase
CRITICAL NOTE — Periodicity of Discrete Sinusoids:
A discrete sinusoidal signal x(n) = A·cos(ω₀n + θ) is periodic with period N if and only if:
ω₀ = 2πk/N for some integers k and N

In other words, ω₀/(2π) must be a rational number (ratio of two integers).

Example 1 — PERIODIC: x(n) = 2cos(0.1πn + π/3)
ω₀ = 0.1π = π/10, so ω₀/(2π) = 1/20 = rational → Period N = 20 ✓

Example 2 — NOT PERIODIC: x(n) = 2cos(2n)
ω₀ = 2 rad/sample, so ω₀/(2π) = 1/π = irrational → Non-periodic ✗

Important: All continuous-time sinusoidal signals are periodic, but discrete-time sinusoidal signals can be either periodic or non-periodic!
📐 MATLAB: Periodic Sinusoidal Sequence

2.4 Even/Odd Synthesis and Energy/Power

Important Theorems in Discrete-Time Signal Theory

1. Unit Sample Synthesis
x(n) = Σ x(k) · δ(n − k)    (sum over all k) Any arbitrary sequence x(n) can be synthesized as a weighted sum of delayed and scaled unit sample sequences
Why This Matters This is the fundamental basis for the convolution sum. If we know how a linear system responds to each individual impulse δ(n − k), we can determine how it responds to any arbitrary input by summing weighted, shifted impulse responses.
2. Even and Odd Synthesis
Even (Symmetric) Sequence A real-valued sequence x_e(n) is called even if: x_e(−n) = x_e(n)
The even component of any signal is: x_e(n) = ½[x(n) + x(−n)]
Odd (Antisymmetric) Sequence A real-valued sequence x_o(n) is called odd if: x_o(−n) = −x_o(n)
The odd component of any signal is: x_o(n) = ½[x(n) − x(−n)]
Any real-valued sequence:   x(n) = x_e(n) + x_o(n) Even Part: x_e(n) = ½ [x(n) + x(−n)] Odd Part: x_o(n) = ½ [x(n) − x(−n)]
Example 1 — Even/Odd Decomposition Let x(n) = u(n) − u(n − 10). Decompose into even and odd components.
Solution — Step by Step Step 1: Identify x(n)
x(n) = u(n) − u(n−10) is a rectangular pulse: x(n) = 1 for 0 ≤ n ≤ 9, and 0 elsewhere.

Step 2: Find x(−n)
x(−n): time-reversed pulse = 1 for −9 ≤ n ≤ 0, and 0 elsewhere.

Step 3: Compute x_e(n) = ½[x(n) + x(−n)]
• For n = 0: x_e(0) = ½[1 + 1] = 1
• For 1 ≤ n ≤ 9: x_e(n) = ½[1 + 0] = 0.5
• For −9 ≤ n ≤ −1: x_e(n) = ½[0 + 1] = 0.5
• Otherwise: x_e(n) = 0
→ x_e(n) is symmetric about n = 0, as expected.

Step 4: Compute x_o(n) = ½[x(n) − x(−n)]
• For n = 0: x_o(0) = ½[1 − 1] = 0
• For 1 ≤ n ≤ 9: x_o(n) = ½[1 − 0] = 0.5
• For −9 ≤ n ≤ −1: x_o(n) = ½[0 − 1] = −0.5
• Otherwise: x_o(n) = 0
→ x_o(n) is antisymmetric, as expected.

Verification: x_e(n) + x_o(n) = x(n) ✓
Example 2 — Find Even/Odd Parts Find even and odd parts of: (a) x(n) = u(n)    (b) x(n) = aⁿu(n)
Solution (a): x(n) = u(n) x(n) = u(n) = 1 for n ≥ 0, 0 for n < 0
x(−n) = u(−n) = 1 for n ≤ 0, 0 for n > 0

Even part:
• n > 0: x_e(n) = ½[1 + 0] = 0.5
• n = 0: x_e(0) = ½[1 + 1] = 1
• n < 0: x_e(n) = ½[0 + 1] = 0.5
So x_e(n) = ½ for n ≠ 0, x_e(0) = 1

Odd part:
• n > 0: x_o(n) = ½[1 − 0] = 0.5
• n = 0: x_o(0) = 0
• n < 0: x_o(n) = ½[0 − 1] = −0.5
So x_o(n) = ½·sgn(n) where sgn(n) = 1 for n>0, 0 for n=0, −1 for n<0
Solution (b): x(n) = aⁿu(n) x(n) = aⁿu(n): x(n) = aⁿ for n ≥ 0, 0 for n < 0
x(−n) = a⁻ⁿu(−n): x(−n) = a⁻ⁿ for n ≤ 0, 0 for n > 0

Even part:
• n > 0: x_e(n) = ½[aⁿ + 0] = ½aⁿ
• n = 0: x_e(0) = ½[1 + 1] = 1
• n < 0: x_e(n) = ½[0 + a⁻ⁿ] = ½a|n|
So x_e(n) = ½a|n| for all n

Odd part:
• n > 0: x_o(n) = ½aⁿ
• n = 0: x_o(0) = 0
• n < 0: x_o(n) = −½a|n|
So x_o(n) = ½·sgn(n)·a|n|
3. Signal Energy
E = Σ |x(n)|²    (sum over all n = −∞ to +∞) If E < ∞ → x(n) is an ENERGY SIGNAL (also called a finite-energy signal) If E = ∞ → x(n) is NOT an energy signal
Example 1 — Calculate Energy Calculate the energy of x[n] = {−1, 0, 2, 3, 1} (finite duration)
Solution E = |−1|² + |0|² + |2|² + |3|² + |1|²
E = 1 + 0 + 4 + 9 + 1
E = 15 (Energy Signal, E is finite)
4. Signal Power
P = lim[N→∞] { 1/(2N+1) · Σ |x(n)|² }   (sum from n = −N to N) If P < ∞ and P ≠ 0 → x(n) is a POWER SIGNAL

For a periodic sequence with fundamental period N:

P = (1/N) · Σ |x(n)|²    (sum over one complete period N)
Example 2 — Calculate Average Power of Periodic Signal x[n] = {…, −1, 0, 2, 4, −1, 0, 2, 4, −1, 0, 2, 4, …} (periodic with N = 4)
Solution Period N = 4. One period: {−1, 0, 2, 4}
P = (1/N) · Σ |x(n)|² over one period
P = (1/4) · [|−1|² + |0|² + |2|² + |4|²]
P = (1/4) · [1 + 0 + 4 + 16]
P = (1/4) · 21
P = 5.25 (Power Signal)

Note: Since the signal is periodic and infinite in duration, its total energy E = ∞ (not an energy signal), but its average power P = 5.25 (finite → it IS a power signal).
Example 3 — Classify x(n) = (3/2)ⁿ u(−n) Find the power of x(n) = (3/2)ⁿ u(−n). Is it an energy signal or power signal?
Solution — Careful Analysis u(−n) = 1 for n ≤ 0, 0 for n > 0. So x(n) = (3/2)ⁿ for n ≤ 0, else 0.

Let's analyze for n ≤ 0. Let m = −n where m ≥ 0:
x(−m) = (3/2)⁻ᵐ = (2/3)ᵐ

So the signal values for n = 0, −1, −2, −3, … are:
x(0) = (3/2)⁰ = 1
x(−1) = (3/2)⁻¹ = 2/3 ≈ 0.667
x(−2) = (3/2)⁻² = (2/3)² ≈ 0.444
x(−3) = (3/2)⁻³ = (2/3)³ ≈ 0.296 …

The signal decays as n → −∞ (since |2/3| < 1).

Energy calculation:
E = Σ_{n=−∞}^{0} |(3/2)ⁿ|² = Σ_{m=0}^{∞} (2/3)^{2m} = Σ_{m=0}^{∞} (4/9)^m
This is a geometric series with ratio r = 4/9 < 1:
E = 1/(1 − 4/9) = 1/(5/9) = 9/5
E = 1.8 (finite) → Energy Signal → Power P = 0

2.5 Discrete-Time Systems — Properties

Discrete-Time System (Definition) A discrete-time system is described as an operator T[·] that takes a sequence x(n) (called the excitation or input) and transforms it into another sequence y(n) (called the response or output):
y(n) = T[x(n)]

Property 1 — Linearity (Superposition)

A system is LINEAR if and only if: T[a·x₁(n) + b·x₂(n)] = a·T[x₁(n)] + b·T[x₂(n)] = a·y₁(n) + b·y₂(n)
How to Test Linearity — Two Sub-Properties:
1. Additivity: T[x₁(n) + x₂(n)] = T[x₁(n)] + T[x₂(n)]
2. Homogeneity (Scaling): T[a·x(n)] = a·T[x(n)]

If BOTH are satisfied → Linear. If EITHER fails → Nonlinear.

Testing Procedure:
  1. Apply input x₁(n) → get output y₁(n) = T[x₁(n)]
  2. Apply input x₂(n) → get output y₂(n) = T[x₂(n)]
  3. Apply input x₃(n) = a·x₁(n) + b·x₂(n) → get output y₃(n)
  4. Check: if y₃(n) = a·y₁(n) + b·y₂(n) → LINEAR; else → NONLINEAR
Worked Example — Test y(n) = x²(n) for Linearity
Solution y₁(n) = T[x₁(n)] = x₁²(n)
y₂(n) = T[x₂(n)] = x₂²(n)
y₃(n) = T[x₁(n) + x₂(n)] = [x₁(n) + x₂(n)]² = x₁²(n) + 2x₁(n)x₂(n) + x₂²(n)

Expected (if linear): y₁(n) + y₂(n) = x₁²(n) + x₂²(n)

Actual: y₃(n) = x₁²(n) + x₂²(n) + 2x₁(n)x₂(n) ≠ y₁(n) + y₂(n) (in general)
→ NONLINEAR (the cross term 2x₁x₂ violates superposition)

Property 2 — Time Invariance (Shift Invariance)

A system is TIME-INVARIANT if: If y(n) = T[x(n)], then T[x(n − k)] = y(n − k) for any integer k "A shift in the input causes an identical shift in the output"
Testing Procedure:
  1. Apply x(n) → get y₁(n) = T[x(n)]; then delay it to get y₁(n − k)
  2. Apply delayed input x(n − k) → get output y₂(n) = T[x(n − k)]
  3. Compare: if y₂(n) = y₁(n − k) → TIME INVARIANT; else → TIME VARIANT
Note: Time invariant systems are also called shift-invariant systems. A key indicator of time variance is when n appears explicitly as a coefficient in the system equation (e.g., n², 2ⁿ).
Worked Example — Test y(n) = n·x(n) for Time Invariance
Solution Path 1: y₁(n) = T[x(n)] = n·x(n); then shift: y₁(n − k) = (n − k)·x(n − k)
Path 2: Shift input first: x(n − k); apply system: y₂(n) = T[x(n − k)] = n·x(n − k)

Compare: y₂(n) = n·x(n − k) ≠ (n − k)·x(n − k) = y₁(n − k)
→ TIME VARIANT (the explicit 'n' coefficient causes the mismatch)

LTI — Linear Time-Invariant Systems

Linear Time-Invariant (LTI) System A linear system in which an input-output pair x(n) and y(n) is invariant to a shift in time. Both linearity AND time-invariance must hold.
LTI systems are completely characterized in the time domain by their impulse response h(n).

Property 3 — BIBO Stability

A system is BIBO STABLE if: |x(n)| ≤ Mx < ∞ (bounded input) implies |y(n)| ≤ My < ∞ (bounded output)
BIBO Stability Condition for LTI Systems:
An LTI system is BIBO stable if and only if its impulse response is absolutely summable:
S = Σ |h(n)| < ∞    (sum over all n)

If S < ∞ → Stable
If S = ∞ → Unstable

Why this matters: An unstable system can cause burnout, saturation, or runaway oscillations in hardware — potentially dangerous and destructive!

Property 4 — Causality

A system is CAUSAL if: The output at time n₀ depends only on inputs at times n ≤ n₀ (Output does NOT depend on future inputs)
Causality Condition for LTI Systems:
An LTI system is causal if and only if:
h(n) = 0   for all n < 0
Such an impulse response is called a causal sequence.

Why it matters: Causal systems can actually be built — they don't need future samples which haven't arrived yet. Unless otherwise stated in DSP, we always assume causal systems.
Classification Problem — Classify y(n) = n²·e^{x(n)} Classify the system y(n) = n²·e^{x(n)} as: Linear/Nonlinear, Time-Invariant/Time-Variant, Causal/Non-Causal
Complete Solution Linearity Test:
T[a·x₁(n) + b·x₂(n)] = n²·e^{a·x₁(n) + b·x₂(n)} = n²·e^{a·x₁(n)}·e^{b·x₂(n)}
a·T[x₁(n)] + b·T[x₂(n)] = a·n²·e^{x₁(n)} + b·n²·e^{x₂(n)}
These are NOT equal → NONLINEAR (the exponential violates superposition)

Time Invariance Test:
Path 1: y₁(n) = n²·e^{x(n)} → delay → y₁(n−k) = (n−k)²·e^{x(n−k)}
Path 2: y₂(n) = T[x(n−k)] = n²·e^{x(n−k)}
y₂(n) = n²·e^{x(n−k)} ≠ (n−k)²·e^{x(n−k)} = y₁(n−k)
→ TIME VARIANT (the n² coefficient shifts differently)

Causality Test:
y(n) = n²·e^{x(n)}: the output at time n depends only on the input at the SAME time n — no future inputs.
→ CAUSAL
Classify: y(n) = ax(n−1) + x(−n)
Solution Linearity: Both ax(n−1) and x(−n) are linear operations. T[αx₁ + βx₂] = αT[x₁] + βT[x₂] ✓ → LINEAR

Time Invariance:
Path 1: y₁(n) = ax(n−1) + x(−n) → y₁(n−k) = ax(n−k−1) + x(−n+k)
Path 2: y₂(n) = T[x(n−k)] = a·x(n−k−1) + x(−(n)−k) = ax(n−k−1) + x(−n−k)
Compare: y₁(n−k) has x(−n+k), but y₂(n) has x(−n−k) — these differ! → TIME VARIANT

Causality: The term x(−n) for n < 0 involves x(|n|) which are future values → NON-CAUSAL
Classify: y(n) = x(n²)
Solution Linearity: T[αx₁+βx₂] = αx₁(n²)+βx₂(n²) = αT[x₁]+βT[x₂] → LINEAR

Time Invariance:
Path 1: y₁(n) = x(n²) → y₁(n−k) = x((n−k)²)
Path 2: y₂(n) = T[x(n−k)] = x((n²)−k) [substituting n by n² then shifting]
(n−k)² = n²−2nk+k² ≠ n²−k → TIME VARIANT

Causality: y(n) = x(n²). For n = 2: y(2) = x(4). We need x(4) to compute y(2) — but n=4 is in the future relative to n=2. → NON-CAUSAL

2.6 Linear Convolution

LTI systems are completely characterized in the time domain by their impulse response h(n). The output y(n) for any arbitrary input x(n) is found using the convolution sum.

Derivation from First Principles:
1. Unit impulse response: δ(n) → [LTI System] → h(n)
2. By time invariance: δ(n−k) → [LTI] → h(n−k)
3. By homogeneity: x(k)·δ(n−k) → [LTI] → x(k)·h(n−k)
4. By additivity (superposition): Σ x(k)·δ(n−k) → [LTI] → Σ x(k)·h(n−k)
5. Since x(n) = Σ x(k)·δ(n−k), it follows that y(n) = Σ x(k)·h(n−k)
y(n) = x(n) * h(n) = Σ x(k) · h(n − k) (sum from k = −∞ to +∞) Also written as: y(n) = Σ h(k) · x(n − k)
Properties of Convolution:
  • Commutative: x(n) * h(n) = h(n) * x(n)
  • Associative: [x(n) * h₁(n)] * h₂(n) = x(n) * [h₁(n) * h₂(n)]
  • Distributive: x(n) * [h₁(n) + h₂(n)] = x(n)*h₁(n) + x(n)*h₂(n)
  • Identity: x(n) * δ(n) = x(n)
  • Length: If x(n) has length M and h(n) has length N, then y(n) = x(n)*h(n) has length L = M + N − 1

Convolution — Manual Computation Methods

Method 1 — The Tabular (Matrix) Method:
Write x(n) horizontally and h(n) vertically. Multiply each x(k) by every h(n−k), then sum along diagonals.

Method 2 — The Sliding Window Method:
Fold h(k) → h(−k), then slide it across x(k). At each position n, compute the dot product Σ x(k)·h(n−k).

Method 3 — Polynomial Multiplication:
Treat x(n) and h(n) as polynomial coefficients. The product of the two polynomials gives y(n).

y(2) = x(0)·h(2) + x(1)·h(1) + x(2)·h(0) = 3·0 + 11·3 + 7·2 = 0 + 33 + 14 = 47

y(n) = {6, 31, 47, 6, −51, −5, 41, 18, −22, −3, 8, 2}

2.7 Difference Equations

An LTI discrete system can be described by a linear constant-coefficient difference equation:

Σ_{k=0}^{N} aₖ · y(n − k) = Σ_{k=0}^{M} bₖ · x(n − k) Or equivalently: y(n) = Σ_{k=0}^{M} bₖ·x(n−k) − Σ_{k=1}^{N} aₖ·y(n−k)
Structure:
• The bₖ terms involve delayed versions of the input x(n) → these form the FIR (Moving Average) part
• The aₖ terms (for k ≥ 1) involve delayed versions of the output y(n) → these form the IIR (Recursive) part
• If only bₖ terms exist → FIR filter; if aₖ terms exist → IIR filter

Chapter 3 — Transform-Domain Analysis: Z-Transform & Fourier

3.1 The Z-Transform

The Fourier Transform of a discrete-time sequence may not converge for all sequences (particularly non-absolutely-summable ones). The Z-Transform is a more general representation that overcomes this limitation and is critical for analyzing discrete-time systems.

Z-Transform (Bilateral/Two-Sided): X(z) = Σ x(n) · z⁻ⁿ    (sum from n = −∞ to +∞) where z is a complex variable: z = |z| · e^{jω}
Inverse Z-Transform: x(n) = (1/2πj) ∮ X(z) · zⁿ⁻¹ dz    (contour integral)

Key Comments on the Z-Transform

  • The complex variable z is called the complex frequency: z = |z|·e^{jω}, where |z| is the attenuation and ω is the real frequency.
  • The function |z| = 1, equivalently z = e^{jω}, is a circle of unit radius in the z-plane — called the unit circle.
  • If the ROC contains the unit circle, X(z) evaluated on the unit circle gives the Discrete-Time Fourier Transform (DTFT): X(e^{jω}) = X(z)|_{z=e^{jω}}
  • Therefore, the DTFT is a special case of the Z-transform evaluated on the unit circle.
  • The ROC (Region of Convergence) is crucial — it defines for which values of z the sum converges, and it determines the unique inverse transform.

Example 1 — Right-Sided (Positive-Time) Sequence Let x₁(n) = aⁿu(n), where 0 < |a| < ∞. Find the Z-Transform and ROC.
Solution — Detailed Derivation X₁(z) = Σ_{n=−∞}^{∞} aⁿu(n)·z⁻ⁿ = Σ_{n=0}^{∞} aⁿ·z⁻ⁿ    [u(n) selects n ≥ 0]
= Σ_{n=0}^{∞} (a/z)ⁿ = Σ_{n=0}^{∞} (az⁻¹)ⁿ

This is a geometric series Σ rⁿ = 1/(1−r), which converges when |r| < 1.
Condition: |az⁻¹| = |a|/|z| < 1 → |z| > |a|

Therefore:
X₁(z) = 1/(1 − az⁻¹) = z/(z − a)    for |z| > |a|

ROC: The exterior of a circle of radius |a| in the z-plane (all z outside the circle).
Example 2 — Left-Sided (Negative-Time) Sequence Let x₂(n) = −bⁿu(−n−1), where 0 < |b| < ∞. Find Z-Transform and ROC.
Solution X₂(z) = Σ_{n=−∞}^{∞} [−bⁿu(−n−1)]·z⁻ⁿ = −Σ_{n=−∞}^{−1} bⁿ·z⁻ⁿ
Let m = −n (so when n goes −1 to −∞, m goes 1 to ∞):
= −Σ_{m=1}^{∞} b⁻ᵐ·zᵐ = −Σ_{m=1}^{∞} (z/b)ᵐ

= −[(z/b)/(1 − z/b)] for |z/b| < 1, i.e., |z| < |b|
= −z/b · b/(b − z) = −z/(b − z) = z/(z − b)

Wait — simplifying: −[(z/b)/(1 − z/b)] = −z/(b−z) = z/(z−b) = 1/(1−bz⁻¹)
X₂(z) = 1/(1 − bz⁻¹) = z/(z − b)    for |z| < |b|

CRITICAL NOTE: If b = a, then X₂(z) = X₁(z) algebraically, but their ROCs are DIFFERENT! ROC₁: |z| > |a| vs ROC₂: |z| < |a|. This proves that the ROC is a DISTINGUISHING feature — the Z-transform pair is NOT unique without specifying the ROC!
Example 3 — Two-Sided Sequence Let x₃(n) = aⁿu(n) − bⁿu(−n−1). Find Z-Transform and ROC.
Solution Using linearity and Examples 1 & 2:
X₃(z) = X₁(z) + X₂(z) = 1/(1−az⁻¹) + 1/(1−bz⁻¹)

But the ROCs must overlap:
ROC₁: |z| > |a| and ROC₂: |z| < |b|
ROC₃ = ROC₁ ∩ ROC₂ = {|a| < |z| < |b|}

Case 1: If |b| < |a| → ROC₃ is empty → X₃(z) does NOT exist!
Case 2: If |a| < |b| → ROC₃: |a| < |z| < |b| (a ring/annulus)
and X₃(z) = 1/(1−az⁻¹) + 1/(1−bz⁻¹) for this ROC.
Example 4 — Finite Right-Sided Sequence x₁(n) = {2, 3, 4} for n = 0, 1, 2 (zero elsewhere). Find Z-Transform and ROC.
Solution X₁(z) = Σ x(n)·z⁻ⁿ = x(0)·z⁰ + x(1)·z⁻¹ + x(2)·z⁻²
= 2·1 + 3·z⁻¹ + 4·z⁻²
X₁(z) = 2 + 3z⁻¹ + 4z⁻²    ROC: Entire Z-plane except z = ∞
(The z⁻ⁿ terms blow up as z → 0 only for negative n; here n ≥ 0, so z = 0 is fine, but z = ∞ makes z⁻¹ = 0, which is fine here actually — ROC is entire plane except possibly z = ∞ due to z⁻¹ terms going to 0 there, so ROC = all z except z = ∞)
Example 5 — Finite Left-Sided Sequence x₂(n) = {2, −1, 3, 0} for n = −3, −2, −1, 0. Find Z-Transform and ROC.
Solution X₂(z) = x(−3)·z³ + x(−2)·z² + x(−1)·z¹ + x(0)·z⁰
= 2z³ + (−1)z² + 3z + 0
X₂(z) = 2z³ − z² + 3z    ROC: Entire Z-plane except z = 0
(The positive powers of z blow up as z → ∞, but that's avoided — the ROC excludes z = 0 due to zⁿ terms for n < 0 becoming z^{negative} = 1/z^{|n|} which blow up at z = 0)
Example 6 — Finite Two-Sided Sequence x₃(n) = {2, −1, 3, 2, 3, 4} for n = −3, −2, −1, 0, 1, 2. Find Z-Transform and ROC.
Solution X₃(z) = 2z³ + (−1)z² + 3z¹ + 2z⁰ + 3z⁻¹ + 4z⁻²
= 2z³ − z² + 3z + 2 + 3z⁻¹ + 4z⁻²
ROC: Entire Z-plane except z = 0 and z = ∞
(Positive powers blow up at z = ∞; negative powers blow up at z = 0)
Example 7 — Standard Z-Transform Pairs Find Z-transform and ROC of: (a) δ(n) (b) u(n) (c) δ(n−1) (d) δ(n+1)
(a) Unit Impulse δ(n) X(z) = Σ δ(n)·z⁻ⁿ = δ(0)·z⁰ = 1·1 = 1
Z{δ(n)} = 1    ROC: Entire Z-plane
(b) Unit Step u(n) X(z) = Σ_{n=0}^{∞} z⁻ⁿ = Σ (z⁻¹)ⁿ = 1/(1 − z⁻¹) for |z| > 1
Z{u(n)} = 1/(1 − z⁻¹) = z/(z−1)    ROC: |z| > 1
(c) Delayed Impulse δ(n−1) X(z) = Σ δ(n−1)·z⁻ⁿ = z⁻¹ [only non-zero at n=1]
Z{δ(n−1)} = z⁻¹    ROC: Entire Z-plane except z = 0
(d) Advanced Impulse δ(n+1) X(z) = Σ δ(n+1)·z⁻ⁿ = z¹ = z [only non-zero at n = −1]
Z{δ(n+1)} = z    ROC: Entire Z-plane except z = ∞

3.3 System Representation in the Z-Domain

System Function H(z): H(z) = Y(z) / X(z) = [Σ_{k=0}^{M} bₖ·z⁻ᵏ] / [Σ_{k=0}^{N} aₖ·z⁻ᵏ] Y(z) = H(z) · X(z)    where ROC_y = ROC_h ∩ ROC_x
What the System Function Tells Us:
  • Poles: Values of z where H(z) → ∞ (denominator = 0). Poles inside the unit circle → stable causal system.
  • Zeros: Values of z where H(z) = 0 (numerator = 0).
  • Frequency response: H(e^{jω}) = H(z)|_{z=e^{jω}} gives the magnitude and phase response as a function of digital frequency ω.
  • Stability: For a causal LTI system, BIBO stable iff all poles are inside the unit circle (|z| < 1).

3.4 The Discrete-Time Fourier Transform (DTFT)

If x(n) is absolutely summable (Σ |x(n)| < ∞), then its DTFT is well-defined:

DTFT: X(e^{jω}) = Σ x(n) · e^{−jωn}    (sum over all n) Inverse DTFT: x(n) = (1/2π) ∫_{−π}^{π} X(e^{jω}) e^{jωn} dω Note: X(e^{jω}) is periodic with period 2π
Key Properties of the DTFT:
  • X(e^{jω}) is a continuous function of ω (even though x(n) is discrete)
  • It is periodic with period 2π
  • The magnitude |X(e^{jω})| is the frequency response / spectrum
  • For real x(n): |X(e^{jω})| is even, ∠X(e^{jω}) is odd
  • Corresponds to Z-transform evaluated on the unit circle
Worked Problem — Find DTFT of x[n] = {1, 0, −1} x[n] = δ(n+1) − δ(n−1) (or equivalently x(0)=0, x(1)=−1, x(−1)=1 — NO: x[n]=[1,0,−1] starting at n=0 means x(0)=1, x(1)=0, x(2)=−1)
Solution — x[n] = {1, 0, −1} starting at n = 0 x(0) = 1, x(1) = 0, x(2) = −1, all other n → 0

X(e^{jω}) = Σ x(n)·e^{−jωn} = x(0)·e⁰ + x(1)·e^{−jω} + x(2)·e^{−j2ω}
= 1 + 0·e^{−jω} + (−1)·e^{−j2ω}
X(e^{jω}) = 1 − e^{−j2ω}

Factor out e^{−jω}:
= e^{−jω}[e^{jω} − e^{−jω}] = e^{−jω} · 2j·sin(ω)

Magnitude: |X(e^{jω})| = 2|sin(ω)|
Phase: ∠X(e^{jω}) = −ω + π/2 (for sin(ω) > 0)

Frequency Response of an LTI System

H(e^{jω}) = H(z)|_{z = e^{jω}} = Y(e^{jω}) / X(e^{jω}) |H(e^{jω})| = Magnitude Response ∠H(e^{jω}) = Phase Response
Worked Problem — y[n] = 0.9y[n−1] + x[n] (Low-Pass System) Determine the frequency response manually and plot using MATLAB.
Manual Solution From difference equation: Y(z) = 0.9z⁻¹Y(z) + X(z)
H(z) = 1 / (1 − 0.9z⁻¹)

Evaluate on unit circle z = e^{jω}:
H(e^{jω}) = 1 / (1 − 0.9e^{−jω})

Magnitude:
|H(e^{jω})| = 1 / |1 − 0.9e^{−jω}|
= 1 / |1 − 0.9cos(ω) + j0.9sin(ω)|
= 1 / √[(1 − 0.9cos(ω))² + (0.9sin(ω))²]
= 1 / √[1 − 1.8cos(ω) + 0.81]
= 1 / √[1.81 − 1.8cos(ω)]

At ω = 0: |H(1)| = 1/√[1.81 − 1.8] = 1/√0.01 = 1/0.1 = 10 (high gain at DC → low-pass!)
At ω = π: |H(−1)| = 1/√[1.81 + 1.8] = 1/√3.61 ≈ 0.527 (low gain at high freq → confirms LP)

Chapter 4 — Implementation of Discrete-Time Systems

We now use the theory from Chapters 2 and 3 for actual signal processing. To process signals, we design and implement systems using basic building blocks mapped from difference equations to block diagrams.

4.1 Basic Elements of DT System Implementation

Adder
Two or more inputs → one output (sum). Addition of three or more signals is done by cascading two-input adders. Symbol: circle with + inside.
Multiplier (Gain)
Single input → single output. Multiplies the signal by a constant coefficient (gain). Multiplication by 1 is implicit and not shown. Symbol: triangle with coefficient.
Delay Element (Shifter / Memory) — z⁻¹ Delays the signal passing through it by exactly one sample. Implemented using a shift register (a memory location that shifts its content each clock cycle). The Z-domain equivalent is multiplication by z⁻¹. Multiple delays = z⁻ᴺ = N delay elements in series.

4.2 IIR System Structures

The general system function of an IIR filter is:

H(z) = [Σ_{k=0}^{M} bₖ·z⁻ᵏ] / [Σ_{k=0}^{N} aₖ·z⁻ᵏ] = B(z)/A(z) with a₀ = 1 (by convention)

The corresponding difference equation:

y(n) = Σ_{k=0}^{M} bₖ·x(n−k) − Σ_{k=1}^{N} aₖ·y(n−k)

4.2.1 Direct Form I Structure

Structure: Implements the difference equation directly as given, in two separate cascaded sections:
  1. Section 1 (Zeros/FIR part): Moving average — implements the numerator B(z). Creates a tapped delay line with M delays feeding into multipliers with coefficients b₀, b₁, …, bₘ. Their weighted sum goes to the output.
  2. Section 2 (Poles/IIR part): Recursive feedback — implements the denominator A(z). Takes the output y(n) through N delays, with feedback multipliers a₁, a₂, …, aₙ summed back to the output.

Number of delay elements: M + N total delay elements (two separate delay lines)
For M = N = 4: requires 4 + 4 = 8 delay elements

4.2.2 Direct Form II (Canonical) Structure

Key Idea: Interchange the order of the two cascade sections in Direct Form I. The two delay lines are now adjacent — connected by a unity gain branch — so one delay line can be eliminated.

Structure:
  1. Input x(n) first goes through the recursive feedback section (poles) to generate an intermediate sequence w(n)
  2. w(n) then goes through the feedforward section (zeros) to produce output y(n)
  3. Both sections SHARE the same delay line
Number of delay elements: max(M, N) — the canonical (minimum) number
For M = N = 4: requires only 4 delay elements (saving 4 compared to Direct Form I)

Both forms are equivalent from input/output perspective; internally they have different signals.

4.2.3 Cascade Form

Method: Factor H(z) into smaller second-order sections called biquads (second-order sections with 2 poles and 2 zeros):
H(z) = b₀ · Π_k H_k(z) where each H_k(z) = (b₀ₖ + b₁ₖz⁻¹ + b₂ₖz⁻²)/(1 + a₁ₖz⁻¹ + a₂ₖz⁻²)

Each biquad section is implemented in Direct Form II (canonical). The entire system is a cascade (series) of biquads.

Advantages: Less sensitive to coefficient quantization errors; easier to control poles/zeros individually.

4.2.4 Parallel Form

Method: Use partial fraction expansion to decompose H(z) into a sum of simpler sections:
H(z) = H₀ + Σ_k Hₖ(z) where each Hₖ(z) is a first-order or second-order section

Each section is implemented in a direct form, and all sections run in PARALLEL — inputs of all sections connected to x(n), outputs summed to produce y(n).

Advantages: Parallel computation (hardware can run sections simultaneously); poles of individual sections are independent.

Lab 7(i) — DFT using DIT (Decimation In Time) FFT Algorithm

Aim: To compute the DFT using the DIT (Cooley-Tukey) FFT butterfly algorithm.

DIT FFT — Theory The FFT (Fast Fourier Transform) computes the N-point DFT in O(N log₂N) operations instead of O(N²) for the direct DFT. The DIT algorithm:
1. Bit-reverse the input sequence (reorder by bit-reversed index)
2. Compute N/2 butterfly operations at each of log₂(N) stages
3. Each butterfly: B = A + W·C, D = A − W·C, where W = e^{−j2πk/N} are twiddle factors
📐 MATLAB Program — DIT FFT
function b = ditdft(a) a = bitrevorder(a); % Step 1: bit-reverse the input N = length(a); l = log2(N); % number of stages for m = 1:1:l % for each stage m % compute twiddle factors for this stage for t = 0:1:((2^(m-1))-1) k(t+1) = (N*t) / 2^m; wn(t+1) = exp((-2*i*pi*k(t+1)) / N); end % butterfly operations for p = 1:2^m:(N - 2^(m-1)) r = 1; x = p; for q = x+2^(m-1) : 1 : (x+2^m)-1 b(p) = a(p) + a(q)*wn(r); % butterfly output 1 b(q) = a(p) - a(q)*wn(r); % butterfly output 2 p = p + 1; r = r + 1; end end a = b; % update for next stage end return; end
Sample Run: a = [1 2 2 2] b = ditdft(a) b = [7 -1 -1 -1] Verification using direct DFT: X(0)=1+2+2+2=7 ✓ X(1) = 1 + 2·e^{-jπ/2} + 2·e^{-jπ} + 2·e^{-j3π/2} = 1 + 2(0-j) + 2(-1) + 2(0+j) = 1-j·2-2+j·2 = -1 ✓

Lab 8(i) — DFT using DIF (Decimation In Frequency) FFT Algorithm

Aim: To compute the DFT using DIF FFT algorithm.

DIF vs DIT: In DIT, bit-reversal is done on the INPUT (before the butterfly stages). In DIF, bit-reversal is done on the OUTPUT (after the butterfly stages). The butterfly structure is also different: in DIF, the twiddle factor multiplication comes AFTER the addition/subtraction.

📐 MATLAB Program — DIF FFT
function a = difdft(a) N = length(a); l = log2(N); for m = l:-1:1 % stages run from l down to 1 (reversed vs DIT) for t = 0:1:((2^(m-1))-1) k(t+1) = (N*t) / 2^m; wn(t+1) = exp((-2*i*pi*k(t+1)) / N); end for p = 1:2^m:(N - 2^(m-1)) r = 1; x = p; for q = x+2^(m-1) : 1 : (x+2^m)-1 b(p) = a(p) + a(q); % add first b(q) = (a(p) - a(q)) * wn(r); % then multiply by twiddle (DIF butterfly) p = p + 1; r = r + 1; end end a = b; end a = bitrevorder(a); % bit-reverse the OUTPUT (DIF specific) return; end

Lab 10 — FIR LPF using Hamming Window

2. Compute the ideal impulse response hd(n) = sin(wc·n)/(π·n)
3. Apply a window function w(n) to truncate the ideal response
4. h(n) = hd(n) · w(n) is the practical FIR filter

Lab 12(i) — Auto-Correlation

Aim: To find the auto-correlation of a given sequence.

Auto-Correlation (Definition) The auto-correlation of x(n) measures the similarity of the signal with a time-shifted version of itself:
r_xx(l) = Σ x(n) · x(n−l) = x(l) * x(−l)
Length of result: 2N − 1 for a length-N sequence
Property: r_xx(l) = r_xx(−l) — auto-correlation is always even-symmetric
Peak: r_xx(0) = Σ x²(n) = signal energy (always maximum at l = 0)

Lab 12(ii) — Cross-Correlation

Aim: To find the cross-correlation between two given sequences.

Cross-Correlation (Definition) The cross-correlation of x(n) and h(n) measures similarity between two different signals:
r_xh(l) = Σ x(n) · h(n−l) = x(l) * h(−l)
Application: Detecting a known signal h(n) within noise-corrupted x(n) — radar, sonar, communications
📐 MATLAB Program — Cross-Correlation

Problem 1 — System Classification Classify y(n) = n²·e^{x(n)} into: Linear/Nonlinear, Time-Variant/Time-Invariant, Causal/Non-Causal
Complete Solution 1. Linearity Test:
Let y₁(n) = n²·e^{x₁(n)} and y₂(n) = n²·e^{x₂(n)}
Apply input: x₃(n) = α·x₁(n) + β·x₂(n)
y₃(n) = T[x₃(n)] = n²·e^{α·x₁(n) + β·x₂(n)} = n²·e^{α·x₁(n)}·e^{β·x₂(n)}
Expected (if linear): α·y₁(n) + β·y₂(n) = αn²·e^{x₁(n)} + βn²·e^{x₂(n)}
y₃(n) ≠ expected (generally) → NONLINEAR ✓
The exponential operation e^{x(n)} violates superposition.

2. Time Invariance Test:
y₁(n) = T[x(n)] = n²·e^{x(n)}
Delay the output: y₁(n−k) = (n−k)²·e^{x(n−k)}
Apply delayed input: y₂(n) = T[x(n−k)] = n²·e^{x(n−k)}
Comparison: y₂(n) = n²·e^{x(n−k)} vs y₁(n−k) = (n−k)²·e^{x(n−k)}
n² ≠ (n−k)² in general → y₂(n) ≠ y₁(n−k) → TIME VARIANT ✓
The coefficient n² shifts differently than the signal.

3. Causality Test:
y(n) = n²·e^{x(n)}: the output at time n depends only on x(n) — the current input value only.
There is no dependence on x(n+1), x(n+2), etc. (no future values)
→ CAUSAL ✓

Problem 5 — Difference Equation Sequence Generation Determine and plot the sequence y(n) = x(n) − α·x(n−1) where x(n) = αⁿu(n).
Solution x(n) = αⁿu(n): values are α⁰=1, α¹, α², α³, … for n = 0, 1, 2, 3, …
x(n−1) = α^{n−1}u(n−1): values are 0 at n=0, α⁰=1 at n=1, α¹ at n=2, …

y(n) = x(n) − α·x(n−1) = αⁿu(n) − α·α^{n−1}u(n−1)
= αⁿu(n) − αⁿu(n−1)
= αⁿ[u(n) − u(n−1)]
= αⁿ·δ(n)    [since u(n) − u(n−1) = δ(n)]
= α⁰·δ(n) = δ(n)
y(n) = δ(n) — a unit impulse!

This is an important result: the first-difference filter applied to a geometric sequence produces an impulse. This shows that y(n) = x(n) − αx(n−1) is an inverse filter for the system that has impulse response αⁿu(n).
